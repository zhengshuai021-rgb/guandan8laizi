# 八红桃掼蛋 — 发牌控制机制说明

> 版本：v2  |  日期：2026-06-29  |  项目：`projects/guandan8laizi/`

---

## 1. 玩法概述

### 经典掼蛋 vs 八红桃

| | 经典掼蛋 | 八红桃掼蛋 |
|---|---|---|
| **万能牌数量** | 2 张（2 张级牌视为红桃♥） | **8 张**（全部 8 张级牌视为红桃♥） |
| **牌库** | 108 张（2 副标准扑克） | 108 张（相同） |
| **每人手牌** | 27 张 | 27 张 |
| **癞子密度** | ~1.9% | ~7.4% |

八红桃的核心变化：癞子数量从 2 → 8，使得手牌中炸弹、同花顺等高级牌型大幅增加，游戏节奏更快、刺激度更高。

## 2. 发牌控制总览

### 两阶段发牌

```
┌──────────────────────────────────────────────────────┐
│                    108 张牌库                          │
│         8 张癞子（级牌） + 100 张自然牌                  │
└──────────────────────────────────────────────────────┘
                          │
          ┌───────────────┴───────────────┐
          ▼                               ▼
    ┌──────────┐                   ┌──────────┐
    │ 阶段一    │                   │ 阶段二    │
    │ 癞子分配  │                   │ 自然牌发放 │
    └──────────┘                   └──────────┘

          ▼                               ▼

          │                               │
          └───────────────┬───────────────┘
                          ▼
                 ┌─────────────────┐
                 │  复用计分评估    │
                 │  癞子同时计入    │
                 │  多种牌型       │
                 └─────────────────┘
```

## 3. 阶段一：癞子分配

### 3.1 概率档位

| 档位 | 癞子数量 | 概率 | 期望贡献 |
|------|----------|------|----------|
| A | 1 ~ 2 张 | **69%** | 1.04 |
| B | 3 ~ 4 张 | **20%** | 0.70 |
| C | 5 ~ 7 张 | **10%** | 0.60 |
| D | 8 张 | **1%** | 0.08 |
| **合计** | | **100%** | **2.42** |

> ⚠️ 原始期望 2.42 × 4 = 9.66 > 8（库存），需要削峰填谷。

### 3.2 分配算法

```
对每位玩家：
  1. 按概率摇档位（A/B/C/D）
  2. 档位内随机取精确值

削峰：如果 4 人总和 > 8
  重复：从癞子最多的人减 1 张

填谷：如果 4 人总和 < 8
  重复：给癞子最少的人加 1 张

最后：4 人癞子数量随机打乱座位
```

### 3.3 分配效果（10 万次模拟）

| 癞子数 | 占比 |
|--------|------|
| 1 张 | 20.8% |
| **2 张** | **62.3%** |
| 3 张 | 13.4% |
| 4 张 | 3.0% |
| 5 张 | 0.5% |

- 83% 的人在 1-2 张区间
- 人均癞子 = 2.0
- 4 人长期均癞子完全一致 ✅

## 4. 阶段二：自然牌发放

### 4.1 策略选项

#### 策略 A：完全随机（基准线）

100 张自然牌洗乱后均匀发给 4 人（每人 25 张，已扣除癞子数）。

- ✅ 简单
- ❌ 单局方差大

#### 策略 B：补偿控制（推荐）

1. 先按策略 A 随机发完
2. 评估 4 人手牌分数
3. 如果最强与最弱差距 > 30%
4. 从最强手随机抽 2~4 张牌补给最弱手

- ✅ 额外开销极小
- ✅ 缩小极端差距
- ⚠️ 调牌数量有限，不改变根本结构

### 4.2 完整手牌构成

每人 **27 张**：
- 癞子（级牌）：由阶段一确定
- 自然牌（100 张中分配）：
  - 每 rank（A~K）：各有 8 张（4 花色 × 2 副）
  - 大王：2 张
  - 小王：2 张

## 5. 牌力评估体系

### 5.1 核心原则：癞子复用计分

**同一张癞子可以同时计入多种牌型。** 这是因为癞子代表"灵活性"——手上有 3 张癞子，意味着你可以选择把它们用在炸弹、同花顺或对子上，虽然实战只能选一种，但**拥有选择权本身就是牌力**。

```
示例：
  自然牌：♠A ♠2 ♠3 ♠4  +  癞子 = 1 张

  复用评估：
  - ♠A ♠2 ♠3 ♠4 + 癞子 → 同花顺 ✅  (1 癞子补 ♠5)
  - 癞子还能和手上其他牌配…

  同一张癞子被计入同花顺、炸弹等多个指标
```

### 5.2 评估维度

| 维度 | 计算规则（含癞子复用） |
|------|----------------------|
| **炸弹** | 每个 rank：自然张数 + 癞子数 ≥ 4 → 计 1 个炸弹 |
| **同花顺** | 每个花色：最长连续 rank + 癞子补断口 ≥ 5 → 计 1 个 |
| **对子** | 每个 rank：自然张数 + 癞子数 ≥ 2 → 计 1 个对子 |
| **三张** | 每个 rank：自然张数 + 癞子数 ≥ 3 → 计 1 个三张 |
| **钢板** | 相邻两个 rank 各 ≥3（含癞子）→ 计 1 个 |
| **连对** | 连续 3+ 个 rank 各 ≥2（含癞子）→ 计 N 个 |
| **大王** | 自然牌中的大王张数 |
| **小王** | 自然牌中的小wang张数 |

### 5.3 综合评分公式

```
score = 炸弹×30 + 同花顺×25 + 钢板×15 + 三张×6 + 对子×2
      + 连对×10 + 大王×10 + 小王×6 + 癞子×5
```

> 权重反映牌型在实战中的价值：炸弹 > 同花顺 > 钢板 > 连对 ≈ 大王 > 三张 > 对子

## 6. 模拟结果

### 6.1 长期均衡性

| 玩家 | 均分 | 标准差 | 结论 |
|------|------|--------|------|
| P1 | ~940 | ~224 | ✅ 均衡 |
| P2 | ~940 | ~226 | ✅ 均衡 |
| P3 | ~940 | ~225 | ✅ 均衡 |
| P4 | ~940 | ~225 | ✅ 均衡 |

4 人长期完全一致（随机座次 + 补偿控制）。

### 6.2 癞子 vs 牌力

| 癞子数 | 出现率 | 均分 | 炸弹(均) | 同花顺(均) |
|--------|--------|------|----------|------------|
| 1 张 | 20.7% | 537 | 5.0 | 5.7 |
| 2 张 | 62.4% | 1016 | 7.7 | 17.6 |

关键发现：**癞子从 1→2 张，牌力翻 2 倍**。2 张癞子补齐同花顺断口的能力是质变。

## 7. 配置参数

所有参数集中在 `deal_v2.py` 顶部，可直接修改：

```python
TIERS = [
    (1, 2, 69),    # 1~2张，69%
    (3, 4, 20),    # 3~4张，20%
    (5, 7, 10),    # 5~7张，10%
    (8, 8, 1),     # 8张，1%
]

TOTAL_LAIZI = 8              # 癞子总数
PLAYERS = 4                  # 玩家数
HAND_SIZE = 27               # 每人手牌数
LEVEL_RANK = "2"             # 当前级牌（默认 2）
CONTROL_MODE = "compensate"  # none | batch | compensate
```

## 10. 已知局限 & 改进方向

1. **补偿力度有限** — 调 4 张牌不改变牌型结构，需要更精准的"按需补牌"策略
2. **同花顺评估偏乐观** — 滑动窗口+癞子复用可能高估实际可成的同花顺数
3. **未考虑搭档配合** — 4 人掼蛋是 2v2 合作，当前只评估个人牌力
4. **牌型权重可调** — 炸弹:同花顺 30:25 的权重可根据实战经验调优
