## 1. 功能概述

八癞子一键理牌将玩家的 27 张手牌（含 0~8 张万能癞子牌）自动编排成最优牌型组合，目标与普通掼蛋一致：**减少单张散牌**、**优先组成炸弹/同花顺/顺子/木板/钢板/三带二**。核心算法位于 `sort_8laizi.py`，枚举上千种候选方案后选择"剩余单牌最少"的结果。

### 与普通掼蛋的关键差异

| 维度 | 普通掼蛋 | 八癞子 |
|------|----------|--------|
| 癞子数量 | 2张（红桃级牌） | 0~8张（全部 4 花色 × 2 副牌 = 8 张级牌） |
| 搜索空间 | ~120 种组合 | ~2000+ 种组合 |
| 新增维度 | 无 | 癞子在炸弹 vs 同花顺之间的分配比例 |
| 同花顺 | 5张同花连续 | 严格 5 张同花连续（不可延长） |
| 策略文件 | `poker.js` k7.game.gd.sort | `sort_8laizi.py` |

## 3. 数据结构

### 3.1 Card（单张牌）

| 字段 | 类型 | 说明 |
|------|------|------|
| `suit` | str | 花色: S/H/C/D/X |
| `rank` | str | 点数: A/2~K/SJ/BJ |
| `is_wild` | bool | 是否为癞子（级牌） |
| `power` | int | 牌点权值（见 6.1） |
| `value` | int | 牌点数值（A=14, 2=2~K=13） |
| `cid` | int | 唯一标识（用于 JSON 序列化后回配） |

### 3.2 CardGroup（一组牌型）

| 字段 | 说明 |
|------|------|
| `cards` | list[Card] |
| `group_type` | king / flush / bomb / straight / board / steel / three_two / triple / pair / single |
| `power` | 牌型权值（见 6.2） |
| `size` | 组内张数 = len(cards) |

### 3.3 SortResult（一种理牌方案的结果）

```python
class SortResult:
    kings: list[CardGroup]         # 王炸
    flushes: list[CardGroup]       # 同花顺
    bombs: list[CardGroup]         # 炸弹
    straights: list[CardGroup]     # 顺子
    boards: list[CardGroup]        # 木板（连对）
    steels: list[CardGroup]        # 钢板（连续三张）
    three_with_twos: list[CardGroup]  # 三带二
    triples: list[CardGroup]       # 三张
    pairs: list[CardGroup]         # 对子
    singles: list[CardGroup]       # 单张
```

### 3.4 三区划分

最终显示按 `炸弹区 → 非理牌区 → 理牌右区` 顺序：

| 区 | 包含牌型 | 分区规则 |
|----|---------|---------|
| 炸弹区 (sortBombs) | 王炸、同花顺、炸弹(4~10线) | `bombs_output` getter: 王炸→同花顺→5+线炸(张数↓, sort_power↓)→4线炸(sort_power↓) |
| 非理牌区 (notsort) | 三张、对子、单张、及其他 | 不在右区的全部归此区，组间按 sort_power↓ |
| 理牌右区 (sortR) | 顺子(5张)、木板(6张)、钢板(6张)、三带二(5张) | 组 size 为 5 或 6 的特定牌型；区内排序: 三带二→顺子→木板→钢板，同类型内按 power↓ |

## 4. 核心算法：策略枚举

### 4.1 三大策略组

| 策略组 | 提取顺序 | 说明 |
|--------|---------|------|
| **O_flush_first** | 王炸 → 同花顺 → 炸弹 → 24排列(顺子/木板/钢板/三带二) → 三张/对子/单张 | 同花顺优先于炸弹 |
| **N_bomb_first** | 王炸 → 炸弹 → 同花顺 → 24排列 → 三张/对子/单张 | 炸弹优先于同花顺 |
| **O_flush_no_straight** | 同 O_flush_first，但去掉"顺子"提取 | 跳过顺子项（18排列） |

### 4.2 癞子分配扫描维度

八癞子相比普通掼蛋新增的关键维度：`bomb_wilds` — 炸弹可使用的癞子上限。

```python
for bomb_wilds in range(n_lz + 1):   # 0 到 全部癞子
    for order in EXTRACTION_ORDERS:   # 24 种排列
        execute_strategy(naturals, wilds, strategy, bomb_wilds, order)
```

`execute_strategy` 内部：
- **方案 O**：先调用 `extract_flush_straights(pool, wp)` 消耗癞子，再调用 `extract_bombs(pool, wp, bomb_wilds)`，炸弹最多用 `bomb_wilds` 个癞子
- **方案 N**：顺序相反，先炸弹后同花顺

总搜索量（以 3 癞子为例）：`3 × 24 + 3 × 24 + 3 × 18 = 198` 种策略。

### 4.3 24 种提取排列

```python
EXTRACTION_ORDERS = list(itertools.permutations(
    ["straight", "board", "steel", "three_two"]
))
# 共 4! = 24 种，如: (straight, board, steel, three_two), (board, three_two, straight, steel), ...
```

每种排列中按顺序贪心提取对应牌型。

## 5. 各牌型提取函数

### 5.1 王炸 (`extract_king_bombs`)

4 张大小王（2副牌共 4 张 Joker）= 1 个王炸，power = 1017。

### 5.2 同花顺 (`extract_flush_straights`)

**严格 5 张同花色连续牌**，癞子补断口。支持 A 作为高牌（10-J-Q-K-A）。

```
10 种可能的 5 卡窗口：
  A-2-3-4-5 (索引 0-4)  ~  9-10-J-Q-K (索引 8-12)  = 9 个普通窗口
  10-J-Q-K-A (索引 9-13, A 视为高牌 13) = 1 个 A 高牌窗口

对每个花色：
  遍历 10 个窗口 → 计算所需癞子数 → 选癞子最少 + 起点最高的窗口
  每 rank 最多取 1 张自然牌 → 不足 5 张用癞子补足
```

**重要约束**：同花顺不容许前后延伸超过 5 张。只从 10 个固定 5 卡窗口中选择。

**Power 公式**：`520 + 首非癞子value + max(0, 4 - 首非癞子索引)`

### 5.3 炸弹 (`extract_bombs`)

分 4 个阶段贪心提取：

| 阶段 | 规则 | 消耗癞子 |
|------|------|---------|
| Phase 0 | 纯自然炸弹（≥4 张同 rank，不消耗癞子） | 0 |
| Phase 1 | 癞子补足：3张→4线, 2张→4线, 1张→4线 | ≤ max_wilds_for_bombs |
| Phase 2 | 已有 4+ 张自然炸弹 + 癞子 = 5+ 线炸弹 | ≤ remaining |
| Phase 3 | 纯癞子炸弹（4 癞子 = 1 炸弹） | 4/个 |

**Power 公式**：`首牌power + 张数 × 100`

### 5.4 顺子 (`extract_straights`)

贪心提 5 张连续（不限花色），癞子补断口。支持 A 低牌 (A-2-3-4-5) 和 A 高牌 (10-J-Q-K-A)。
每次选癞子最少 + 长度最短的连续段。

**Power 公式**：`首牌power + 顺子长度 - 1`

### 5.5 木板 / 连对 (`extract_boards`)

≥3 个连续 rank 各有 ≥2 张（含癞子补足）。贪心找最长连续段。

### 5.6 钢板 (`extract_steel_plates`)

≥2 个连续 rank 各有 ≥3 张（含癞子补足）。贪心找最长连续段。每 rank 最多取 3 张自然牌。

### 5.7 三带二 (`extract_three_with_two`)

三张 + 对子，对子点数 ≤ J（`max_pair_value=11`）。贪心取可用三张 + 可用对子。

### 5.8 残余提取 (`extract_remaining`)

三张 → 对子 → 单张，按 rank 点数降序 + 张数降序贪心。

## 6. 排序规则

### 6.1 单牌 power（牌点权值）

| 牌 | power | 说明 |
|---|-------|------|
| 2~K | `value` | 2=2, ..., K=13 |
| A | 14 | 特殊：A 的 value 即 14 |
| 癞子（级牌） | 15 | `WILD_POWER` |
| 小王 | 16 | `SMALL_JOKER_POWER` |
| 大王 | 17 | `BIG_JOKER_POWER` |

**顺序**：普通牌 < A(14) < 癞子(15) < 小王(16) < 大王(17)

### 6.2 牌组 group.power（牌型权值）

| 牌型 | power 公式 |
|------|-----------|
| 王炸 | `1017` |
| 同花顺 | `520 + (首非癞子value + max(0, 4 - 首非癞子索引))` |
| 炸弹(4~10线) | `首牌power + 张数 × 100` |
| 顺子 | `首牌power + 顺子长度 - 1` |
| 三带二 / 三张 / 对子 / 单张 | `首牌power` |

### 6.3 sort_power（组间显示排序权值）

```python
def sort_power(g):
    for i, c in enumerate(g.cards):
        if not c.is_wild:
            return c.power + i      # 首非癞子power + 前置癞子偏移
    return g.cards[0].power         # 全癞子取首张
```

`+ i` 补偿前置癞子：癞子排在组前部，越靠前 i 越大，权值越高。

## 7. 方案优选规则

所有候选方案按 `score` 元组排序，越小越好：

```python
def score():
    return (
        len(singles),           # ① 单张数（优先最少）
        -len(bombs),            # ② 炸弹数（越多越好）
        -len(flushes),          # ③ 同花顺数
        -bomb5plus_count,       # ④ 5+线炸弹数
        -len(straights),        # ⑤ 顺子数
        -len(boards),           # ⑥ 木板数
        -len(steels),           # ⑦ 钢板数
        -len(three_with_twos),  # ⑧ 三带二数
        -len(triples),          # ⑨ 三张数
        -len(pairs),            # ⑩ 对子数
    )
```

**优先级**：单牌少 > 炸弹多 > 同花顺多 > 5+线炸多 > 顺子多 > 木板多 > 钢板多 > 三带二多 > 三张多 > 对子多
