## 1. 功能概述

一键理牌将玩家的 27 张手牌自动编排成最优牌型组合，目标是**减少单张散牌**、**优先组成炸弹/同花顺/顺子/木板/钢板/三带二**。核心算法位于 `k7.game.gd.sort`，尝试多种方案后选择"剩余单牌最少"的结果。

## 3. 数据结构：三个分区

CardManager 维护三组牌，最终按 `炸弹区 → 非理牌区 → 理牌右区` 顺序显示。

| 字段 | 类型 | 含义 | 区内排序规则 |
|------|------|------|------------|
| `_sortBombs` | `HandCard[][]` | 炸弹区：王炸/同花顺/普通炸弹(4~10线) | getter 重排：王炸 → 同花顺 → 普通炸弹；普通炸弹按`(张数↓, getSortPower↓)` |
| `_notsort` | `HandCard[][]` | 非理牌区：单张/对子/三张/<5张散牌 | getter 重排：组内按花色↓；组间按 getSortPower↓（sortmix getter） |
| `_sortR` | `HandCard[][]` | 理牌右区：顺子(5张)/木板(6张)/钢板(6张)/三带二(5张) | getter 触发 `groupBySortR()`：固定顺序 Hulu→Link→DoubleLink→TripleLink |

**最终显示顺序**（`handcards` getter + `drawCards`）：
```ts
// CardManager.ts:130-132
public get handcards(): HandCard[][] {
    return this.sortBombs.concat(this.sortmix, this.sortR);
}
```

## 4. 核心算法：CardManager.quicksort

**位置**：`CardManager.ts:339-375`

```ts
public quicksort(): void {
    // ① 收集"非理牌区"所有牌
    const handcards: HandCard[] = [];
    this.notsort.forEach(group => handcards.push(...group.slice()));
    const volist = this.getVoFromHand(handcards);  // 转 CardVO[]

    // ② 调用 poker.js 核心算法
    // 第二参数 cardValue=11 限制三带二中"二"的点数不超过J
    const { bombs, others } = k7.game.gd.sort(volist, 11);

    // ③ 清空非理牌区，按结果重新分流
    this._notsort = [];

    // 炸弹 → _sortBombs
    for (let i = 0; i < bombs.length; i++) {
        const group = [];
        bombs[i].cards.forEach(c => {
            const idx = handcards.findIndex(h => h.vo.id == c.id);
            const hc = handcards.splice(idx, 1)[0];
            hc && group.push(hc);
        });
        this._sortBombs.push(group);
    }

    // 非炸弹 → 按长度分流
    for (let i = 0; i < others.length; i++) {
        const tmpHands = [];
        others[i].cards.forEach(c => {
            const idx = handcards.findIndex(h => h.vo.id == c.id);
            const hc = handcards.splice(idx, 1)[0];
            hc && tmpHands.push(hc);
        });
        // 5/6 张进右区（顺子/三带二/木板/钢板），其余进非理牌区
        if (tmpHands.length == 5 || tmpHands.length == 6) {
            this._sortR.push(tmpHands);
        } else {
            this._notsort.push(tmpHands);
        }
    }
}
```

**要点**：
- 只对**非理牌区**重排，不清空已有的 `_sortBombs`/`_sortR`（增量式）
- `gd.sort` 返回的 `bombs` 全部进 `_sortBombs`；`others` 按 5/6 张分流
- HandCard ↔ CardVO 通过 `vo.id` 回配

## 5. 核心算法：k7.game.gd.sort

**位置**：`poker.js:1752-1800`（声明见 `poker.d.ts:298`）

### 5.1 方案生成

汇总 **5 种候选方案**，按"剩余单牌最少优先"选择最优解：

| 方案 | 提取顺序 |
|-----|---------|
| sortO 方案1 | 王炸 → 同花顺 → 顺子 → 10~4线炸 → 3张 → 对子 → 单张 |
| sortO 方案2 | 去掉顺子 |
| sortN 方案1 | 王炸 → 同花顺 → 4炸 → 10~5线炸 |
| sortN 方案2 | 王炸 → 4炸 → 同花顺 → 10~5线炸 |
| sortN 方案3 | 王炸 → 10~4线炸 → 同花顺 |

**N 方案内层**：顺子/木板/钢板/三带二 的提取顺序有 **24 种全排列**（`4!`），每种后接单张/对子/三张提取。

### 5.2 方案优选规则

```js
results.sort((a, b) => {
    // 优先级：单牌少 > 炸弹多 > 同花顺多 > 5+线炸多 > 顺子多 > 木板多 > 钢板多 > 三带二多 > 三张多 > 对子多
    if (a.one.length != b.one.length) return a.one.length - b.one.length;
    if (a.bomb4.length != b.bomb4.length) return b.bomb4.length - a.bomb4.length;
    if (a.ths.length != b.ths.length) return b.ths.length - a.ths.length;
    if (a.bomb5.length != b.bomb5.length) return b.bomb5.length - a.bomb5.length;
    if (a.shunzi.length != b.shunzi.length) return b.shunzi.length - a.shunzi.length;
    if (a.muban.length != b.muban.length) return b.muban.length - a.muban.length;
    if (a.gangban.length != b.gangban.length) return b.gangban.length - a.gangban.length;
    if (a.threeWithTwo.length != b.threeWithTwo.length) return b.threeWithTwo.length - a.threeWithTwo.length;
    if (a.three.length != b.three.length) return b.three.length - a.three.length;
    if (a.two.length != b.two.length) return b.two.length - a.two.length;
    return 0;
});
```

### 5.3 返回结构

```js
const bombs  = result.king.concat(result.ths, result.bomb5, result.bomb4);   // 王炸 > 同花顺 > 5-10线炸 > 4线炸
const others = result.shunzi.concat(result.muban, result.gangban,
                                     result.three, result.two, result.one, result.threeWithTwo);
others.sort((a,b) => getSortPower(b.cards) - getSortPower(a.cards));        // 组间按 getSortPower ↓
return { bombs, others };
```

## 6. 排序规则

### 6.1 单牌 power（牌点权值）

| 牌 | value | power | 来源 |
|---|-------|-------|------|
| 2~K | 2~13 | `power = value` | poker.js `initCardsVO` |
| A | 1 | `power = 14` | poker.js `initCardsVO` |
| 级牌/癞子 | 1~13 | `power = 15` | GameComp 覆盖，`MAIN_CARD_POWER` |
| 小王 | 14 | `power = 16` | poker.js `initCardsVO` |
| 大王 | 15 | `power = 17` | poker.js `initCardsVO` |

**顺序**：普通牌 < A(14) < 级牌/癞子(15) < 小王(16) < 大王(17)

### 6.2 牌组 group.power（牌型权值）

用于牌型比较、组间排序：

| 牌型 | power 公式 |
|------|-----------|
| 王炸 | `1017`（`FOUR_KING_POWER`） |
| 同花顺 | `520 + (首非癞子value + (4 - 首非癞子索引))` |
| 炸弹(4~10线) | `首牌power + 张数 * 100` |
| 顺子 | `首牌power + 顺子长度 - 1` |
| 三带二 | `三张部分首牌power` |
| 三张/对子/单张 | `首牌power` |

### 6.3 getSortPower（组间显示排序权值）

用于 `sortBombs`、`sortmix`、`others` 的组间排序：

```ts
// CardManager.ts:321-325 / poker.js 内联
private getSortPower(hand: HandCard[]): number {
    const cards = this.getVoFromHand(hand);
    const i = cards.findIndex(c => !c.isWild);          // 首个非癞子索引
    return i == -1 ? cards[0].power : cards[i].power + i; // 全癞子取首张；否则 首非癞子power + 前置癞子数
}
```

`+ i` 补偿前置癞子：癞子排在组前部，越靠前 i 越大，权值越高。

### 6.4 区间排序规则

| 区 | 区之间 | 区内组间 | 区内组内 |
|----|--------|---------|---------|
| 炸弹区 | 王炸 > 同花顺 > 普通炸弹 | 普通炸弹：`(张数↓, getSortPower↓)`；同花顺：`getSortPower↓` | 普通炸弹：癞子集中前，同类按花色↓ |
| 非理牌区 | 固定在炸弹区之后、右区之前 | `getSortPower↓` | 按花色↓（黑>红>梅>方） |
| 理牌右区 | 固定最后 | Hulu → Link → DoubleLink → TripleLink；同牌型 `group.power↓` | Hulu/木板/钢板：距 power 距离；顺子：不重排 |
